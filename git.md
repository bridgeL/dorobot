# Git 提交规范

## 允许多次提交

允许多次提交，让各个提交涉及到的范围更加合理

## 提交信息格式

提交信息应该简明扼要，聚焦在**做了什么**。

不要在commit信息中记录改了什么文件， git diff已有此功能，不需要在commit信息中重复编写。

**格式：** `[emoji] type: 简短描述`

1. **必须使用前缀** - 所有提交都必须包含 `[emoji] type:` 前缀
2. **描述功能，而非文件** - 提交信息应该说明你实现了什么，而不是改了哪些文件
3. **说明原因，而非过程** - 修复 bug 时要解释 bug 产生的原因，而不是描述你如何一行行改代码
4. **量化收益** - 重构时简要说明带来的改进（性能、可维护性等）
5. **一句话表述** - 用最少的话表达清楚意思，避免冗长描述

## 提交类型前缀

| 类型 | Emoji | 说明 | 示例 |
|------|-------|------|------|
| `feat` | ✨ | 新功能 | `✨ feat: 实现用户登录功能` |
| `fix` | 🐛 | 修复 Bug | `🐛 fix: 修复用户重复注册的问题` |
| `refactor` | ♻️ | 重构代码 | `♻️ refactor: 重构数据库连接池管理` |
| `docs` | 📝 | 文档更新 | `📝 docs: 编写项目启动功能的文档` |
| `perf` | ⚡ | 性能优化 | `⚡ perf: 优化消息处理速度` |
| `test` | ✅ | 测试相关 | `✅ test: 添加用户模块单元测试` |
| `chore` | 🔧 | 构建/工具 | `🔧 chore: 更新依赖版本` |
| `style` | 🎨 | 代码格式 | `🎨 style: 统一代码缩进风格` |
| `ci` | 👷 | CI/CD 相关 | `👷 ci: 添加自动化部署配置` |

## 正确示例

### 功能实现

```
✨ feat: 实现用户登录功能

支持邮箱和密码验证，成功登录后返回 JWT Token
```

### Bug 修复

```
🐛 fix: 修复用户重复注册的问题

原因：并发请求时未加锁，导致同时插入相同邮箱的账号
```

### 重构

```
♻️ refactor: 重构数据库连接池管理

提升：连接复用率提高 40%，减少高并发下的连接超时问题
```

### 文档

```
📝 docs: 编写项目启动功能的文档
```

### 性能优化

```
⚡ perf: 优化消息广播机制

提升：减少 50% 的内存拷贝，支持 10000+ 并发连接
```

## 提交信息结构

```
<emoji> <type>: <简短标题（一句话描述）>

<详细说明（可选，需要时补充背景、原因、收益）>
```

标题必须清晰，详细说明只在必要时添加。如果改动较小，例如代码/文档改动不超过20行，应该不用写详细说明。
