# 犯人在跳舞 — 状态机设计文档

## 一、插件生命周期状态图

```
┌─────────────────────────────────────────────────────────────────┐
│                         插件生命周期                               │
│                                                                 │
│   [未激活] ──(有人发送/开始)──> [room] ──(人数>=3)──> [game]      │
│                                         │                        │
│                                         │人数不足时提示            │
│                                         ↓                        │
│   [结束] <──(游戏结束)────────────────[game.play]                 │
│              ↑                                                  │
│              └──(无人离开时)──[room]                             │
```

## 二、详细状态定义

### 2.1 room 状态 — 等待玩家加入

| 属性 | 类型 | 说明 |
|------|------|------|
| `host` | str | 启动游戏的玩家 ID |
| `players` | list[Player] | 当前房间内的玩家列表 |

**响应命令**：
| 命令 | 触发条件 | 行为 |
|------|----------|------|
| `/加入` | 任意用户发送 | 将发送者加入 `players` |
| `/离开` | 已在房间的玩家发送 | 将发送者从 `players` 移除 |
| `/房间` | 任意用户发送 | 回复当前房间成员列表 |
| `/开始` | 任意用户发送 | 当 `len(players) >= 3` 时进入 `game.play` 状态，否则提示人数不足 |

**退出条件**：
- 调用 `/开始` 且人数 >= 3 → 进入 `game.play`
- 房间为空（所有玩家离开）→ 自动关闭插件

---

### 2.2 game 子状态 — 出牌阶段

所有 game 子状态共用以下属性：

| 属性 | 类型 | 说明 |
|------|------|------|
| `players` | list[Player] | 所有参与玩家（按回合顺序） |
| `hands` | dict[str, list[Card]] | 玩家 ID → 手牌列表 |
| `turn` | int | 当前轮到第几个玩家（0-index） |
| `current_player` | str | 当前出牌玩家 ID |
| `first_card_played` | bool | 第一张牌是否已打出 |
| `private_sessions` | dict[str, str] | 玩家 ID → 私聊 session_id（用于注册私聊监听） |

#### 2.2.1 game.play — 正常出牌阶段

**响应**：
| 来源 | 内容 | 行为 |
|------|------|------|
| 私聊 | `/手牌` | 回复发送者的手牌列表 |
| 群聊 | `/状态` | 公布所有人的手牌数、当前回合等信息 |
| 私聊 | `[卡牌名]` | 检查并打出该牌，处理效果，切换状态 |

**出牌前检查**：
1. 检查是否是当前玩家
2. 检查手牌是否包含该牌
3. 检查出牌限制：
   - 第一张必须是「第一发现人」
   - 「犯人」仅当手牌≤1时能打出
   - 「侦探/警部」仅当手牌≤2时能打出
   - 「交易」：出牌人必须有手牌，否则无效果打出
   - 「神犬/目击者/侦探/警部」：目标无手牌则无效果打出
4. 执行卡牌效果
5. 判定胜负
6. 切换到下一个玩家

---

#### 2.2.2 game.trade — 交易阶段

| 额外属性 | 类型 | 说明 |
|------|------|------|
| `trade_initiator` | str | 发起交易的玩家 ID |
| `trade_target` | str | 交易目标玩家 ID |
| `initiator_card` | str | 发起者选择的卡牌（等待中为 None） |
| `target_card` | str | 目标选择的卡牌（等待中为 None） |

**限制**：
- 出牌人必须有手牌，否则无效果打出
- 目标无手牌则此牌失效，不进入交易状态

**流程**：
```
game.play (打出交易牌，有效)
  → game.trade
    → 双方各自在私聊中选择一张牌
    → 两张牌交换
  → game.play (回到正常出牌)
```

**退出条件**：
- 双方都选择卡牌后交换，回到 `game.play`

---

#### 2.2.3 game.dog — 神犬阶段

| 额外属性 | 类型 | 说明 |
|------|------|------|
| `dog_user` | str | 使用神犬的玩家 ID |
| `dog_target` | str | 神犬目标玩家 ID |

**流程**：
```
game.play (打出神犬牌)
  → game.dog
    → 使用者在群聊中指定目标玩家
    → 目标在私聊中选择弃置的卡牌
  → 弃牌生效，判定胜负
  → game.play
```

**响应**：
| 来源 | 内容 | 行为 |
|------|------|------|
| 群聊 | `[玩家名]` | 使用者指定目标 |
| 私聊 | `[卡牌名]` | 目标选择弃置的卡牌 |

---

#### 2.2.5 game.exchange — 情报交换阶段

| 额外属性 | 类型 | 说明 |
|------|------|------|
| `exchanging_cards` | dict[str, str] | 玩家 ID → 选择的卡牌（等待中为 None） |

**流程**：
```
game.play (打出情报交换牌)
  → game.exchange
    → 所有有手牌的玩家各自在私聊中选择一张牌
    → 按顺时针传递
  → game.play
```

**响应**：
| 来源 | 内容 | 行为 |
|------|------|------|
| 私聊 | `[卡牌名]` | 该玩家选择要交换的卡牌 |

**退出条件**：
- 所有玩家都选择后，按顺时针传递，回到 `game.play`

---

### 2.3 ended 状态 — 游戏结束

| 属性 | 类型 | 说明 |
|------|------|------|
| `winner` | str | 获胜阵营 `"good"` / `"bad"` |
| `message` | str | 结束原因描述 |

**行为**：
- 发送最终结算消息到群聊
- 清除所有私聊监听（`unmount_from_all`）
- 插件自动关闭

## 四、消息监听机制

### 4.1 注册私聊监听

游戏开始时，将每个玩家的私聊 session 与插件绑定，使得私聊消息能转发到游戏：

```python
# 在 game.play 状态开始时
for player in game.players:
    # 获取玩家的私聊 session_id（通过 bot API 获取用户私聊会话）
    private_session_id = await bot.get_private_session(player.id)
    # 挂载到 Layer 1
    plugin.mount_to(private_session_id)
```

### 4.2 清除监听

游戏结束时：

```python
# 在 ended 状态
plugin.unmount_from_all()
```

## 六、胜负判定

| 条件 | 结果 |
|------|------|
| 打出「犯人」牌，无人监视 | 坏人获胜 |
| 打出「犯人」牌，被「警部」监视 | 好人获胜 |
| 「侦探」质疑到持有「犯人」（无不在场证明） | 好人获胜；质疑者是坏人时转为污点证人 |
| 「神犬」使目标弃掉「犯人」牌 | 好人获胜；使用者是坏人时转为污点证人 |
| 「侦探」质疑失败 | 游戏继续 |
| 坏人使用「侦探/神犬/警部」成功抓犯人 | 好人阵营获胜，坏人变为污点证人 |

## 七、状态转换图（game 内部）

```
room ───(人数>=3, /开始)───> game.play
                              │
                              ├──(打出交易牌)──> game.trade ──(双方选完)──┐
                              │                                           │
                              ├──(打出情报交换)──> game.exchange ─(全部选完)┤
                              │                                           │
                              └──(打出神犬牌)──> game.dog ──(目标弃牌)─────┘
                                                               │
                                                               ▼
                                                          game.play
```

每个子状态处理完成后都回到 `game.play`，进入下一轮出牌。
