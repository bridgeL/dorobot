# 犯人在跳舞 — 游戏逻辑文档

## 一、基础规则

**玩家人数**：3-8人

**卡牌分配**：
- 游戏开始时随机发牌，每个玩家轮流出牌
- **第一张打出的牌必须是「第一发现人」**，由该玩家开启回合
- 持「第一发现人」牌的玩家成为第一回合的起始玩家

**回合流程**：按顺序轮到每位玩家出一张牌（可跳过无手牌的玩家），先打完手牌的玩家获胜

---

## 二、阵营与身份

| 阵营 | 身份卡 | 说明 |
|------|--------|------|
| **坏人阵营** | 犯人、共犯 | 坏人阵营获胜条件：打出「犯人」牌并成功逃脱 |
| **好人阵营** | 普通人、侦探、警部、神犬 | 好人阵营获胜条件：抓到此局「犯人」 |

**身份获得时机**：
- **共犯**打出时立即加入坏人阵营（`is_bad = True`）
- **犯人**打出时根据情况决定胜负

---

## 三、卡牌详解

### 3.1 身份牌

| 卡牌 | 名称 | 限制 | 效果 |
|------|------|------|------|
| 第一发现人 | 第一发现人 | 无 | 必须是全局第一张打出的牌 |
| 犯人 | 犯人 | 仅当手牌≤1时能打出 | 打出后坏人获胜；若手牌被质疑则好人获胜 |
| 侦探 | 侦探 | 手牌≤2，需指定目标，目标无手牌则无效果 | 质疑目标——若对方有犯人且无不在场证明，好人获胜；若自己是坏人并成功抓犯人，可作为污点证人转为好人阵营 |
| 警部 | 警部 | 手牌≤2，需指定目标，可对自己使用，目标无手牌则无效果 | 指定玩家——若该玩家最终打出犯人牌，好人获胜；若自己是坏人并成功抓犯人，可作为污点证人转为好人阵营 |
| 不在场证明 | 不在场证明 | 无 | 声明型卡牌——被质疑时若当前手牌中仍有不在场证明则质疑失败 |
| 共犯 | 共犯 | 无 | 打出后立即加入坏人阵营 |
| 普通人 | 普通人 | 无 | 无效果 |
| 神犬 | 神犬 | 需指定目标，目标无手牌则无效果 | 目标弃一张牌——若弃的是犯人牌，好人获胜；若自己是坏人并成功抓犯人，可作为污点证人转为好人阵营；否则目标获得神犬 |

### 3.2 情报/交换牌

| 卡牌 | 名称 | 限制 | 效果 |
|------|------|------|------|
| 谣言 | 谣言 | **无手牌的玩家不参与循环** | 所有有手牌的玩家随机抽一张下家的手牌 |
| 情报交换 | 情报交换 | **无手牌的玩家不参与循环** | 所有有手牌的玩家将一张牌传给他上家 |
| 交易 | 交易 | 需指定另一名玩家；出牌人无手牌则无效果打出，目标无手牌则此牌失效 | 与目标玩家互相交换一张手牌 |
| 目击者 | 目击者 | 需指定目标，目标无手牌则无效果 | 查看目标玩家的手牌 |

## 四、胜负判定

| 场景 | 结果 |
|------|------|
| 打出「犯人」牌，且无人监视 | 坏人获胜 |
| 打出「犯人」牌，被「警部」监视 | 好人获胜 |
| 「侦探」质疑到持有「犯人」（当前手牌中无「不在场证明」）的玩家 | 好人获胜；若质疑者是坏人，可转为污点证人（变为好人）并获胜 |
| 「神犬」使目标弃掉「犯人」牌 | 好人获胜；若神犬主人是坏人，可转为污点证人（变为好人）并获胜 |
| 「侦探」质疑失败（对方无犯人或当前手牌中有不在场证明） | 游戏继续 |
| 坏人使用「侦探」/「神犬」/「警部」成功抓犯人 | 好人阵营获胜，坏人变为污点证人（转为好人） |

### 4.1 污点证人规则

- **初始身份**：所有人默认是好人
- **成为坏人**：打出「共犯」牌后加入坏人阵营
- **污点证人**：坏人使用「侦探」「神犬」「警部」成功抓到犯人时，坏人可以转变为污点证人（重新加入好人阵营），好人阵营获胜
- **被抓的人一定是坏人**：被成功质疑/神犬弃牌/警部监视的犯人，一定是坏人阵营

---

## 五、游戏流程

1. **初始化** → 发牌，确定第一发现人玩家
2. **第一回合** → 第一发现人先出牌（必须为第一发现人牌）
3. **轮流出牌** → 依次轮到每位有手牌的玩家
4. **出牌阶段** → 玩家打出一张牌（部分牌需要选择目标）
5. **子状态交互** → 「交易」「情报交换」「神犬」会切换游戏子状态，进入多阶段交互
6. **游戏结束** → 满足胜负条件时游戏结束

## 六、代码架构

### 6.1 核心类

- **`Game`**：游戏主类，管理游戏状态、玩家、回合流程
- **`Player`**：玩家类，管理玩家手牌和状态
- **`Card`**：卡牌基类，定义卡牌的基本属性和行为
- **`Msg`**：消息基类，用于游戏内通信

### 6.2 卡牌分类

```
cards/
├── first.py       # 第一发现人
├── criminal.py    # 犯人
├── detector.py    # 侦探
├── police.py      # 警部
├── not_me.py      # 不在场证明
├── bad_guy.py     # 共犯
├── dog.py         # 神犬
├── watcher.py     # 目击者
├── rumor.py       # 谣言
├── exchange_info.py # 情报交换
├── trade.py       # 交易
└── simple.py      # 普通人
```

### 6.3 子状态交互

部分卡牌会触发游戏子状态切换，进入多阶段交互：

- **`game.trade`**（交易牌）：等待交易双方各自选择一张牌后交换
- **`game.exchange`**（情报交换牌）：等待所有参与者各自选择一张牌后按顺时针传递
- **`game.dog`**（神犬牌）：等待使用者指定目标，目标再选择一张牌弃掉
